# 勾配ベクトルの生成（gradient noise の方向サンプリング）

Perlin noise（gradient noise）では、格子の各頂点に **疑似ランダムな単位ベクトル（勾配）** を割り当てる。
この「ランダムな向きをどう作るか」のメモ。とくに **2次元から3次元以上へ拡張するとき**に
出てくる落とし穴（向きの一様性）を整理する。

補間側の話は [interpolation.md](./interpolation.md)、座標からランダム値を引く部分の話は
[random-access-prng.md](./random-access-prng.md) を参照。

## TL;DR

- 勾配に求められる性質は **「単位長」「球面上で一様」「格子点ごとに無相関な疑似ランダム」「seed で再現可能」**。
- 2D は `角度 = 2π·hash` → `(cos, sin)` で **円周上一様**になり簡単。実装は
  [`PerlinNoise.grad()`](../src/algorithm/noise/perlin-noise.ts)。
- **3D 以上で「角度を増やすだけ」はダメ**。球面座標で角度を一様に振ると `sin` 重みを無視し、**極に偏る**。
- **回転行列で作るのも楽にならない**。オイラー角を一様に振ると SO(3) の Haar 測度（`sin β` 重み）を
  無視することになり、結局**球面座標と同じ偏り**が同じ形で出る。しかも高次元ほど不利。
- **一様乱数を成分にして正規化してもダメ**。立方体を正規化することになり、**角（対角）方向に偏る**。
- 本命は次元非依存な 2 つ:
  1. **ガウス乱数を n 個引いて正規化**（球対称性で自動的に一様。Muller 法）
  2. **固定勾配セットから選ぶ**（Perlin 原典の 12 本。trig 不要で速い）

---

## 勾配に求められる性質

| 性質 | 理由 |
| --- | --- |
| 単位長（長さ 1） | 内積 `grad·dist` のスケールを揃える。頂点で noise=0 になる性質も保つ |
| 球面上で一様 | 特定方向に偏ると、ノイズに軸沿い・対角沿いのスジ（方向性アーティファクト）が出る |
| 格子点ごとに無相関 | 隣接点で似た向きだと模様が滑らかに繋がりノイズにならない（→ ハッシュ関数の役目） |
| seed で再現可能 | 同じ座標 + seed なら必ず同じ向き |

「無相関・再現可能」は [hash function](./random-access-prng.md) が担保する。本ドキュメントの主題は
残りの **「単位長 × 球面一様」をどう作るか**。

---

## 2次元はかんたん（現状の実装）

```ts
// PerlinNoise.grad() : 円周 [0, 2π) 上に一様な単位ベクトル
private grad(pos: Vector2D): Vector2D {
  const v = this.hashFunction(pos); // [0,1)
  const rad = 2 * Math.PI * v;
  return { x: Math.cos(rad), y: Math.sin(rad) };
}
```

角度 1 個を一様に振るだけで円周上一様になる。**2D は角度の一様 = 向きの一様**なので何も問題がない。

---

## なぜ 3D で「角度を増やすだけ」ではダメか

「2 個目の角度を足して球面座標 `(θ, φ)` にすればいい」と考えがちだが、**球面座標の角度を一様に
振ると極に偏る**。球面の面積要素が

```
dA = sin(θ) dθ dφ
```

と **`sin θ` の重み**を持つため。`θ` を一様に取ると、面積の狭い極付近に点が集まる。

正しくは **`cos θ` を一様**に取る（＝高さ `z` を `[-1, 1]` 一様に取る）:

```ts
// 3D: 球面上で一様な単位ベクトル（ハッシュ値 2 個）
const z   = 2 * v1 - 1;            // cosθ を一様に
const phi = 2 * Math.PI * v2;      // 方位角は一様でよい
const r   = Math.sqrt(1 - z * z);
return { x: r * Math.cos(phi), y: r * Math.sin(phi), z };
```

この `z`+`φ` 法は **3D 専用としては最小コスト**（ハッシュ 2 個 + trig 数回）。

---

## 回転行列で作るのはどうか → 楽にならない

「固定ベクトルを回転行列で回せば向きが作れる」という発想は、**一様な回転を掛ければ**正しい。
だが素直なやり方は次の 3 つの理由で得をしない。

### 罠1: 一様性の問題が同じ形で再発する

オイラー角 `(α, β, γ)` を各々一様に振って回転行列を組んでも、**一様な回転（SO(3) の Haar 測度）に
ならない**。Haar 測度はオイラー角(ZYZ)で

```
dμ ∝ sin(β) dα dβ dγ
```

と **`sin β` 重み**を持つ。これは球面座標の `sin θ` と**まったく同じ Jacobian**。直し方も同じで
`cos β` を一様にする（＝先の `z` を一様に取るのと同一）。**回り道して同じ場所に戻るだけ**。

### 罠2: コストが増える

ちゃんと一様にやるなら一様クォータニオン（Shoemake 法、一様乱数 3 個）→ 回転、という手順になる。

| 方式 | 必要なハッシュ値 | 主な演算 |
| --- | --- | --- |
| 直接サンプル（`z`+`φ`） | 2 個 | trig 数回 |
| 一様クォータニオン → 回転 | 3 個 | quat 生成 + 行列化 + 行列×ベクトル |

結果は直接法と同じで、行列を組んで掛ける分だけ重い。

### 罠3: 高次元でスケールしない

n 次元回転 SO(n) は独立な回転平面が **n(n−1)/2 個**（4D で 6、5D で 10 …）。一様サンプリングは
次元とともに急速に複雑化する。回転は高次元化に対して**スケールしないアプローチ**。

---

## 一様乱数を成分にして正規化 → 角に偏る

「n 個の乱数を取り出して正規化」という手順は正しいが、**一様乱数のまま**やると
「立方体 `[-1,1]ⁿ` 内の一様点を正規化」することになり、**立方体の角（対角線）方向に偏る**。

中心からの距離が方向で違うのが原因:

| 方向 | 中心からの距離 |
| --- | --- |
| 面の中心 | `1` |
| 角（対角） | `√n`（2D で √2、3D で √3） |

正規化は radial に潰す操作なので、立体角あたりの確率が `R(方向)ⁿ` に比例し、線分の長い角方向ほど
点が集まる。

- 2D … 対角(45°,135°…)が軸方向の **約 2 倍**
- 3D … 立方体の **8 隅の対角**にクランプ

「立方体を球だと思って潰している」のが間違いの本質。

---

## ガウス乱数を成分にして正規化 → ちょうど一様（本命）

n 個の独立な正規乱数の**同時密度**は

```
∝ exp(−(x₁² + … + xₙ²) / 2) = exp(−r² / 2)
```

で、**長さ `r` だけの関数 = 回転対称（球対称）**。向きと長さが独立になり、向きは球面上で完全に一様。
だから正規化すれば **次元によらず**一様になる。魔法は正規化ではなく **成分がガウスであること（球対称性）**。

```ts
// n 次元・一様な単位勾配ベクトル（次元に依らず同じコード）
const g = Array.from({ length: n }, () => gaussian()); // 正規乱数 n 個
const len = Math.hypot(...g);
return g.map((c) => c / len);
```

ガウス乱数はハッシュの一様値 2 個から **Box–Muller** で作れる:

```ts
// 一様値 u1,u2 ∈ (0,1] → 独立な標準正規 2 個
const r  = Math.sqrt(-2 * Math.log(u1));
const z0 = r * Math.cos(2 * Math.PI * u2);
const z1 = r * Math.sin(2 * Math.PI * u2);
```

---

## 一様乱数で済ませたいなら：棄却法

一様乱数を使いたい場合は、**単位球の外を捨てて**球対称にすれば一様にできる。

```ts
// [-1,1]ⁿ の一様点を取り、単位球の外なら引き直す
let g, len2;
do {
  g = Array.from({ length: n }, () => 2 * hashUniform() - 1);
  len2 = g.reduce((s, c) => s + c * c, 0);
} while (len2 > 1 || len2 === 0); // 立方体 → 単位球に切り出す
return g.map((c) => c / Math.sqrt(len2));
```

球に切り出してから正規化するので一様。ただし **引く回数が可変**（採用率 = 球/立方体の体積比で
3D で約 52%、高次元でほぼ 0）。**ハッシュの読み取り回数を固定したい**決定論的ノイズとは相性が悪い。

## 実用的な近道：Perlin の固定勾配セット

Ken Perlin の improved noise（2002）は乱数方向を作らず、**立方体の各辺方向に対応する 12 本の固定
勾配**からハッシュで 1 本選ぶ。

```
(±1,±1, 0), (±1, 0,±1), ( 0,±1,±1)   // 12 本（辺の中点方向）
```

- trig を一切使わず**分岐だけで速い**
- 方向が散らばっていて軸沿いの偏りも避けられる
- 厳密な球面一様ではないが、見た目・速度・実装の単純さのバランスがよく原典もこれ

高次元でも同様に「適度に散った固定方向の集合から選ぶ」流儀が使える。

## まとめ（方法の比較）

| 方法 | 球面一様？ | ハッシュ読み取り | 備考 |
| --- | --- | --- | --- |
| 2D：`角度 = 2π·hash` → (cos,sin) | ✓（円周） | 1 個 | 2D 専用・最小コスト |
| 球面座標で角度を一様 | ✗ 極に偏る | 2 個 | `sin θ` 重みを無視した誤り |
| 3D：`z`+`φ` 法（`cos θ` 一様） | ✓ | 2 個 | 3D 専用としては最小コスト |
| 回転行列（オイラー角一様） | ✗ | 3 個 | Haar 測度の `sin β` を無視・球面座標と同じ罠 |
| 回転行列（一様クォータニオン） | ✓ | 3 個 | 正しいが直接法より重い・高次元で破綻 |
| **一様乱数を成分 → 正規化** | ✗ 角に偏る | 固定 n 個 | 立方体を潰している |
| **ガウス乱数を成分 → 正規化** | ✓ | 固定（Box–Muller） | **次元非依存の本命** |
| 一様 + 単位球で棄却 → 正規化 | ✓ | 可変（高次元で激増） | 決定論ノイズと相性悪 |
| 固定勾配セット（Perlin 12 本） | △ 近似 | 1 個（選ぶだけ） | 速い・原典・実用本命 |

**結論**:

- 2D は現状どおり角度 1 個でよい。
- 3D 専用なら `z`+`φ` 法が最小コスト。
- 任意次元へきれいに拡張したいなら **ガウス成分 + 正規化**。
- 速度重視・実用なら **Perlin の固定勾配**。
- **回転行列／一様乱数の素直な拡張は罠**（どちらも同じ「立方体・角度を球と取り違える」偏り）。
